Understand RAG Visually

This interactive simulator demonstrates how a modern Retrieval-Augmented Generation (RAG) system works step-by-step — from document ingestion and chunking to embeddings, vector search, retrieval, prompt augmentation, and final LLM response generation.

📘 RAG Pipeline Steps

1. Document Ingestion

Raw documents are loaded into the system.

2. Chunking

Large documents are broken into smaller chunks.

3. Embeddings

Chunks are converted into vector embeddings.

4. Vector Database

Embeddings are stored for similarity search.

5. Retrieval

Most relevant chunks are retrieved using semantic similarity.

6. Generation

The LLM generates a final grounded answer.
